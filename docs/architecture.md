## Security Model

Kage implements defense-in-depth for command execution:

```
AI Output вҶ’ Scope Validator вҶ’ Safe Mode Filter вҶ’ User Approval вҶ’ Execution вҶ’ Audit Log
```

### Scope Validation
- All targets in commands are validated against the defined scope
- IP addresses, CIDR ranges, and domains are checked
- Excluded targets are blocked

### Safe Mode
- Dangerous commands are blocked (rm -rf, dd, format, etc.)
- Destructive flags are detected
- Network exfiltration patterns are flagged

### Audit Logging
- All actions are logged with timestamps
- Hash chain provides tamper detection
- Session can be replayed for review

## Data Flow

1. **User Input** вҶ’ CLI parses and validates input
2. **Conversation** вҶ’ Context aggregated and sent to LLM
3. **AI Response** вҶ’ Parsed for commands or information
4. **Security Check** вҶ’ Scope + Safe Mode + Approval
5. **Execution** вҶ’ Command runs in configured environment
6. **Results** вҶ’ Output parsed and stored in session
7. **Reporting** вҶ’ Findings compiled into reports
